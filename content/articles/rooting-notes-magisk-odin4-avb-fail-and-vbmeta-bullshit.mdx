# 3. The Disabled VBMETA Experiment

The obvious nerd-brain idea:

```txt
Maybe disabled vbmeta fixes it.
```

Spoiler: it did not. But here’s the process anyway, because pain should be documented.

Create work folders:

```bash
cd ~/Documents/A515F

mkdir -p advanced_root/{stock_vbmeta,patched_ap}
```

Extract VBMETA from the same stock AP:

```bash
cd ~/Documents/A515F/advanced_root/stock_vbmeta

tar -xf ../../firmware/AP_A515FXXU8HWI1_QB71281625_REV00_user_low_ship_meta_OS13.tar.md5 \
  vbmeta.img.lz4 \
  vbmeta_samsung.img.lz4
```

Decompress:

```bash
lz4 -d -f vbmeta.img.lz4 vbmeta.img
lz4 -d -f vbmeta_samsung.img.lz4 vbmeta_samsung.img
```

Patch AVB flags:

```bash
python3 - <<'PY'
from pathlib import Path
import struct

for name in ["vbmeta.img", "vbmeta_samsung.img"]:
    p = Path(name)
    data = bytearray(p.read_bytes())

    if data[:4] != b"AVB0":
        raise SystemExit(f"[!] {name}: not AVB0 vbmeta image")

    old_flags = struct.unpack(">I", data[120:124])[0]
    new_flags = old_flags | 3

    data[120:124] = struct.pack(">I", new_flags)
    p.write_bytes(data)

    print(f"[+] {name}: flags {old_flags} -> {new_flags}")
PY
```

Output:

```txt
[+] vbmeta.img: flags 0 -> 3
[+] vbmeta_samsung.img: flags 0 -> 3
```

Very hacker. Very fancy. Still didn’t mean shit.

Recompress properly for Odin4:

```bash
lz4 -B6 --content-size -f vbmeta.img vbmeta.img.lz4
lz4 -B6 --content-size -f vbmeta_samsung.img vbmeta_samsung.img.lz4
```

Why?

Because if you use plain `lz4 -f`, Odin4 might cry like this:

```txt
Contens size is not defined
Fail parse
```

Yes, “Contens.” Even the error message is broken. Beautiful.

# Technical Takeaways

## 1. Magisk patched AP can trigger CUSTOM VBMETA

On this firmware:

```txt
A515FXXU8HWI1
```

flashing Magisk patched AP can cause:

```txt
ODIN MODE (AVB Fail)
CUSTOM VBMETA
```

Even when Odin4 says the flash completed fine.

Because apparently flashing fine and booting fine are two completely different fucking religions.

---

## 2. Disabled VBMETA did not work here

Setting:

```txt
HASHTREE_DISABLED
VERIFICATION_DISABLED
```

on both:

```txt
vbmeta.img
vbmeta_samsung.img
```

still resulted in:

```txt
ODIN MODE (AVB Fail)
CUSTOM VBMETA
```

So no, “just disable vbmeta bro” was not the magic spell.

---

## 3. Odin4 is picky with LZ4

If you repack `.lz4` files for Odin4, use:

```bash
lz4 -B6 --content-size -f input.img output.img.lz4
```

Otherwise Odin4 may throw:

```txt
Contens size is not defined
Fail parse
```

The typo makes it even worse. Peak Linux flashing experience.
